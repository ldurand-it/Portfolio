Cisco Router CLI

Router On Stick​

int gig0/0.20 -> Le .numéro correspond au numéro de VLAN par convention (c’est une virtualisation d’interface)
encapsulation dot1Q 20 -> Le numéro correspond au numéro de VLAN
ip address 192.168.2.1 255.255.255.0
no sh -> ! bien penser à taper cette commande car bien souvent on oublie
d’allumer l’interface ce qui a pour conséquence de mettre en panne le réseau
pour une erreur bête !

DHCP Relais / IP Helper​

int gig0/1 -> Correspond à l’interface sur laquelle nous souhaitons diffuser le
DHCP
ip helper-address 10.255.255.253 -> L’IP correspond à l’adresse ip de notre
serveur DHCP

RIP​

en
conf t
router rip
version 2
network XXX.XXX.XXX.XXX

Transmettre la route par défaut​

en
conf t
router rip
version 2
default information originate

ACL​

ACL Standard​

R1(config)#access-list 1 deny 192.168.1.100 -> deny permet d’interdire l’adresse suivante
R1(config)#access-list 1 permit 192.168.1.0 0.0.0.255 -> permit autoriser

ACL Étendue​

ip access-list extended David-HTTP
Router(config-ext-nacl)#
R1(config-std-nacl)#permit 192.168.0.0 0.0.0.255
R1(config-std-nacl)#permit 192.168.1.0 0.0.0.255
R1(config-std-nacl)#deny 192.168.0.0 0.0.3.255

NAT / PAT​

Définir un port dans le réseau privé

R1(config)#int fa0/0
R1(config-if)#ip nat inside
R1(config-if)#exit

Définir un port dans le réseau publique

R1(config)#int s0/0
R1(config-if)#ip nat outside
R1(config-if)#exit

NAT Dynamic​

Créer le Pool d'adresses​

R1(config)#ip nat pool POOL-NAT-LAN2 201.49.10.17 201.49.10.30 netmask 255.255.255.240

NAT Static​

R1(config)#ip nat inside source static ”192.168.1.100” “201.49.10.30”
