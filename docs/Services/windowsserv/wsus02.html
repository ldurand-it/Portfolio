Joindre des Ordinateurs à WSUS

Introduction & Prérequis​

Maintenant que vous avez déployé et intégré avec succès WSUS sur un serveur Windows Server 2022, il est temps de faire remonter vos machines du domaine dans votre console WSUS. Pour cette tâche essentielle, nous allons utiliser des stratégies de groupe (GPO).

Ce guide vous accompagne à travers toutes les étapes nécessaires pour intégrer efficacement vos machines à WSUS en utilisant des GPO.

Suivez attentivement ces instructions pour garantir une gestion optimale des mises à jour dans votre environnement informatique. 🛠️

Prérequis​

Avant de commencer le processus d'intégration des ordinateurs à WSUS, assurez-vous de disposer des éléments suivants :

  • Un serveur Windows fonctionnant avec le rôle WSUS installé et configuré. Voir Guide
  • Une connaissance de base de l'utilisation des stratégies de groupe (GPO) dans un environnement Active Directory. Voir Guide
  • Accès administratif au domaine Active Directory pour créer et appliquer des GPO.
  • Les droits nécessaires pour installer des fonctionnalités sur le serveur WSUS.

Intégration des Ordinateurs​

Modifier la Méthode d'Affectation des Ordinateurs​

Pour intégrer nos ortdianteurs dans notre instance WSUS nous allons utiliser les GPO. On va aussi trier 2 Types de Machines ; Les Postes Clients & Les Serveurs.
Pour ça dans WSUS nous allons créer deux Groupes. Dans votre console WSUS à gauche dans la section Tous les ordinateurs cliquez droit et créer donc deux nouveaux Groupes.
Aprrès ça vous devez avoir un résultat similaire au mien.

Ensuite on va définir la méthode d'affectation que WSUS doit utiliser pour recenser nos machines.
Cela se définit dans les Options dans la catégorie Computers et séléctionnez l'oprtion Use Group Policy or registry settings on computers.

Installation des Outils AMDX 11 (Facultatif)​

L'étape qui suit est facultatif car elle n'a pas d'impact fonctionnelle sur nos futurs GPO mais elle permet de simplifier la visualisation des paramètres à utiliser pour nos GPO en les classsant dans des dossiers. Pour mettre à jour les outils AMDX il faut télécharger la dernière version sur le site de Microsoft : https://www.microsoft.com/en-us/download/details.aspx?id=105667
Après les avoirs télécharger sur notre Windows Server 2022 qui gère les GPO donc le controlleur de domaine. On éxécute le fichier et on suit le Wizard d'installation !
Puis on se rend à ce chemin :
C:\Program Files (x86)\Microsoft Group Policy\Windows 11 Month Year Update (XXHX)\PolicyDefinitions
A l'aide des touches CTRL & SHIFT seléctionnez les fichiers suivants :

  • "fr-fr"
  • "en-US"
  • Tous les fichiers avec l'extension .amdx

Copiez les et collez les dans le chemin \\WINSRVO1\sysvol\company.infra\Policies\PolicyDefinitions

INFOS
  • Adaptez le chemin en fonction de votre Infra
  • Créer le Dossier PolicyDefinitions si il n'éxiste pas !

Voilà nous avons installez les templates d'administration ce qui nous permmetra d'y voir plus clair dans nos paramétrages de GPO !

Création des GPO​

Nous voilà maintenant dans le vif du Sujet on va désormais créer nos GPO. Les GPO que nous alons créer sont les suivantes :

  • GPO Pour définir les paramètre communs à tous les Ordinateurs du Domaines que ce soit des PC Clients ou bien des Serveurs
  • GPO Spécifique aux Postes Clients qui définira le Groupe WSUS dans lequel les postes doivent remonter et qui spécifira les heures d'activités de ces machines
  • GPO Spécifique aux Serveurs qui définira le Groupe WSUS dans lequel les serveurs doivent remonter et qui spécifira les heures d'activités de ces machines
GPO-Serveurs

Pour la GPO Serveurs elle ne sera pas détaillé dans ce guide car il suffit de faire un copier coller de celle pour les PC et d'éditez le groupe de Destination ainsi que les heures d'activités

GPO - Paramètres Généraux​

Commencons par la GPO principale qui définira les paramètres suivants :

  • Définition de l'adresse de notre serveur WSUS.
  • Définition des paramètres de mises à jour automatiques.
  • Désactiver la connexion au serveur Microsoft de Windows Update

Pour définir ces paramètres créer une nouvelle GPO et nommez la explicitement (Ex : WSUS - Main Settings) Ensuite modifiez la de cette manière :

Configuration ordinateur > Stratégies > Modèles d’administration > Composants Windows > Windows Update > Gérer les mises à jour proposées de Windows Server Update Service > Spécifier l’emplacement intranet du service de mise à jour Microsoft

Configuration ordinateur > Stratégies > Modèles d’administration > Composants Windows > Windows Update > Gérer l’expérience utilisateur final > Configuration du service Mises à jour automatique

Configuration ordinateur > Stratégies > Modèles d’administration > Composants Windows > Windows Update > Gérer les mises à jour proposées de Windows Server Update Service > Ne pas se connecter à des emplacements Internet Windows Update

GPO - Postes Clients​

Poursuivons maintenant avec la GPO qui va définir les paramètres suivants pour les postes clients :

  • Définition du Groupe WSUS dans lesquels les machines doivents remonter.
  • Définition des heures d'activité de cette catégorie de machines.

Pour définir ces paramètres créer une nouvelle GPO et nommez la explicitement (Ex : WSUS - PC) Ensuite modifiez la de cette manière :

Configuration ordinateur > Stratégies > Modèles d’administration > Composants Windows > Windows Update > Gérer les mises à jour proposées de Windows Server Update Service > Autoriser le ciblage côté client

Configuration ordinateur > Stratégies > Modèles d’administration > Composants Windows > Windows Update > Gérer l’expérience utilisateur final > Désactiver le redémarrage automatique pour les mises à jour pendant les heures d’activité

Test des GPO​

Désormais aprés avoir créer nos GPO on va les appliquez au UO correspondantes et les activer et les tester !
Pour ça on va prendre un poste client et faire un gpupdate /force.
Ensuite on redémarre notre poste et on se rend dans les paramètres du PC et faire un check Updates pour le forcer à aller chercher des mises à jours donc à contacter notre WSUS.

Désormais sur notre Console WSUS on doit voir apparaître un Poste Client !

Résultat​

Bien Joué ! ✔️

Félicitations ! Vous avez maintenant configuré avec succès l'intégration de vos machines dans WSUS à l'aide de GPO sur un serveur Windows Server 2022. En suivant ce guide, vous avez établi un processus efficace pour gérer les mises à jour de manière centralisée dans votre infrastructure.

warning

N'oubliez pas de surveiller régulièrement votre console WSUS pour vous assurer que toutes les machines sont correctement synchronisées et que les mises à jour sont déployées selon vos règles et politiques de sécurité. Avec cette intégration, vous êtes prêt à maintenir vos systèmes à jour et à garantir leur sécurité et leur fiabilité. 🚀
