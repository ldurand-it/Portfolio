Hébergement Web avec les CT de PVE

Introduction & Prérequis​

Bienvenue sur cette documentation.Dans ce guide nous allons installer un serveur Web en utilisant la technologie des conteneurs sur Proxmox.
Dans le but d'héberger un Wiki & le CMS WordPress pour un futur portfolio.

Mise au Point
  • Proxmox VE est une plateforme de virtualisation open-source qui combine la virtualisation de machines (VMs) et la virtualisation par conteneurs dans une interface web conviviale.
  • Nous utiliserons dans ce guide les conteneurs et pas les Machines Virtuelles. Le Conteneur étant plus léger en se basant sur la base de PVE (Debian 12)
note

Dans ce guide nous nous baserons sur Doku Wiki & WordPress avec un template pré-configuré mais n'hésitez pas à utiliser un template vierge pour y héberger vos sites Webs avec un serveur LAMP

Prérequis​

Pour ce guide vous aurez évidemment besoin d'un serveur Proxmox fonctionnel et à jour ainsi que le template à télécharger. Pour le téléchargement du template, rendez vous sur votre volume de stockage (local) et selectionnez cliquez sur template. Cherchez : word Et selectionez turnkey-wordpress
Illustration en Images :

Evidemment on clique sur Download

Création du Conteneur​

Créer un conteneur avec le nom que vous voulez tout en vous assurant qu'il respecte les conventions de nommages définis dans votre environnement.
Définissez un Password sécurisé (Utilisation d'un KeePass) Ensuite les paramètres sont les suivants :

  • Network : en fonction de votre infra
  • Disque Dur : 10GO minimum
  • RAM : 512MIB minimum
  • DNS : en fonction de votre infra

Après avoir configuré votre conteneur vérifiez le et séléctionnez start after created

Utiliation du Conteneur​

SSH​

Logiciels SSH

Pour prendre la main à distance sur votre conteneur utliser un logiciel de gestions de connexion SSH :

  • Tabby, MobaXTerm, Putty, CMD

Prenez l'IP de votre conteneur en vous connectant sur la console et avec ip a Connectez vous en SSH après sur le Port 22 (par défaut) D'ailleurs changez le après pour des questions de sécurité

Prise en Main à Distance

Pour une meilleur utilisation et gestion de votre conteneur l'utilsiationd du SSH est recommandé.

  • En fonction des template l'user Root est utilisé en ssh ou non. Pour changer cela aller dans /etc/ssh/ et modifier le sshd_config

Gestions des sauvegardes​

Les SAUVEGARDES

Pour centralisez vos sauvegades et les planifier vous pouvez utiliser PBS (Proxmox Backup Server) qui sera parfait pour vos conteneur et VM qui tournent sur PVE.

Pour éviter de perdre votre travail sur vos futurs conteneurs faites des sauvegardes avant chaque changement majeur pour être sûr de pouvoir revenir en arrière.

Surveillance de votre Conteneur​

Dans le même principe que les machines virtuelles surveillez régulièrement l'utiisation des ressources de votre conteneur.

Installation du Doku Wiki​

Installation du Serveur Web Apache + PHP​

  • Pour installer Apache & PHP éxécuter les commandes suivantes :
apt update
apt install apache2 libapache2-mod-php php-xml
  • Activer le module
a2enmod userdir
systemctl restart apache2
Premier Test

Vérifier que votre serveur Web est bien en place & accessible.

  • Dans votre navigateur : http://MACHINE-IP
  • Vous arrivez sur une page par défaut de votre Web Server ✔️

Installation Wiki​

  • Placez vous dans le répertoire /var/www
  • Télécharger l'archive
  • Décompressez l'archive
  • Supprimer l'archive
cd /var/www
wget https://download.dokuwiki.org/src/dokuwiki/dokuwiki-stable.tgz
tar xvf dokuwiki-stable.tgz
rm dokuwiki-stable.tgz
  • Changez le proporiétaire
chown -R www-data:www-data /var/www/dokuwiki
  • Éditez le fichier /etc/apache2/sites-available/000-default.conf, enregistrez le sous dokuwiki.conf
Editeur Nano
  • Ctrl + S Sauvegarder
  • Ctrl + X Quittez l'éditeur
  • Ctrl + O Enregistrez sous un autre nom
  • Decommentez la ligne ServerName et ajoutez votre nom DNS wiki_nom.nomdedomaine.local et remplacez la ligne DocumentRoot /var/www/html par DocumentRoot /var/www/dokuwiki.
  • Activez le site du wiki
a2ensite dokuwiki.conf
a2dissite wordpress.conf
  • Redémmarez votre service Apache systemctl restart apache2
Test Wiki

Pour valider la mise en place de votre Wiki rendez vous sur http://adresse_IP_serveur_wiki/install.php

Fichier PHP

Après avoir installer votre serveur DokuWiki supprimer le install.php

Configuration WordPress​

  • Éditez le fichier /etc/apache2/sites-available/wordpress.conf, ajoutez la ligne ServerName wordpress_nom.nomdedomaine.local, enregistrez.
  • Activez le site
a2ensite wordpress.conf
a2dissite dokuwiki.conf
  • Redémmarez votre service Apache systemctl restart apache2
Test WordPress

Pour valider la mise en place de WordPress
Rendez vous avec votre navigateur sur http://adresse_IP_serveur_wordpress/wp-admin

Conclusion​

Bien Joué ! ✔️

Félicitations ! Vous avez maintenant déployé avec succès un conteneur web sur Proxmox VE.
Avec un Service WordPress & DokuWiki !
